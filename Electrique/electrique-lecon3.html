📘 1. Les bascules (élément de base)

Une bascule est un circuit séquentiel capable de stocker 1 bit d'information. C'est l'élément fondamental des compteurs.

Bascule JK

Table de vérité — Bascule JK (sur front descendant)
  J   K   Q(n+1)
  0   0   Q(n)  ← mémorisation
  0   1   0     ← mise à 0 (reset)
  1   0   1     ← mise à 1 (set)
  1   1   Q̄(n)  ← toggle (basculement)

Bascule D (Data/Latch)

Bascule D
Q(n+1) = D   (la sortie suit l'entrée D après un front d'horloge)
🔔

Horloge (CLK) : synchronise les changements d'état

⬆️

Front montant : changement sur le passage de 0→1

⬇️

Front descendant : changement sur le passage de 1→0

🔒

Entrées asynchrones (Preset/Clear) : forcent l'état hors de l'horloge

📗 2. Compteurs asynchrones (ripple counters)

Dans un compteur asynchrone, seule la première bascule reçoit l'horloge. Les bascules suivantes sont pilotées par la sortie de la précédente.

Compteur asynchrone modulo N
Nombre de bascules : n = ⌈log₂ N⌉

⚠️ Inconvénient des compteurs asynchrones

Propagation du signal d'horloge → délai de propagation cumulatif

→ Les états intermédiaires créent des états parasites ("glitchs")

→ Limité aux basses fréquences

📙 3. Compteurs synchrones

Dans un compteur synchrone, toutes les bascules reçoivent le même signal d'horloge. Les entrées J et K (ou D) sont calculées pour produire la séquence désirée.

✅

Avantage : pas de délai de propagation — tous les bits changent simultanément

⚡

Fréquence plus élevée possible

🎯

États parasites éliminés

Logique de commutation synchrone

Pour un compteur binaire, les entrées J/K de chaque bascule sont :

Équations pour compteur binaire synchrone
J₀ = K₀ = 1 (Q₀ bascule à chaque front — bit de poids faible)
J₁ = K₁ = Q₀ (Q₁ bascule quand Q₀=1)
J₂ = K₂ = Q₀·Q₁ (Q₂ bascule quand Q₀=1 ET Q₁=1)

📕 4. Décompteurs et compteurs réversibles

Décompteur

Un décompteur compte à l'envers (7, 6, 5, 4, 3, 2, 1, 0, 7…). La différence avec un compteur :

Différence compteur / décompteur
Compteur : Q₀ commute à chaque front, Q₁ commute quand Q₀ passe de 1→0
Décompteur : Q₁ commute quand Q₀ passe de 0→1

Compteur-décompteur réversible

Mode de fonctionnement
UP = 1 : comptage ↑   |   DOWN = 1 : décomptage ↓

Remise à zéro et préchargement

🔄

Clear (MR) : remise à 0 asynchrone de toutes les bascules

⬆️

Load / Preset : précharge une valeur initiale

📒 5. Diviseurs de fréquence

Un compteur est aussi un diviseur de fréquence : la fréquence de sortie de chaque bascule est divisée par 2.

Division de fréquence
f(Q0) = f_H / 2   |   f(Q1) = f_H / 4   |   f(Qn) = f_H / 2ⁿ

💡 Exemple

Horloge à 1 MHz (f_H = 1 000 000 Hz)

Q0 → 500 kHz   |   Q1 → 250 kHz   |   Q2 → 125 kHz

Q3 → 62.5 kHz   |   Q19 → ≈ 0.95 Hz (visible à l'œil nu !)

⚠️ Compteur modulo N personnalisé

Pour créer un compteur modulo 10 (0→9), on détecte l'état 10 (1010₂) et on active le Clear :

→ Sortie NAND des bascules 1 et 3 → vers l'entrée Clear de toutes les bascules

Circuit intégré courant : 74HC90 (modulo 2 et 5), 74HC161 (modulo 16 synchrone)
