📘 1. Variables et portes logiques

En logique combinatoire, les variables ne prennent que 2 valeurs : 0 (FAUX) ou 1 (VRAI). C'est la base de l'électronique numérique.

🔌

NON (NOT) : S = ā — inverse la valeur

🔌

ET (AND) : S = a · b — résultat = 1 si a ET b = 1

🔌

OU (OR) : S = a + b — résultat = 1 si a OU b = 1

🔌

OU Exclusif (XOR) : S = a ⊕ b — résultat = 1 si a ≠ b

Symboles et équations

  └───┘        b─┤   │S=a·b  b─┤   │       b─┤   │
                 └───┘        └───┘          └───┘

📗 2. Algèbre de Boole

L'algèbre de Boole définit les règles de manipulation des expressions logiques.

Propriétés fondamentales
  Identités       : a + 0 = a     | a · 1 = a
  Élément nul     : a + 1 = 1     | a · 0 = 0
  Idempotence     : a + a = a     | a · a = a
  Complémentaire  : a + ā = 1     | a · ā = 0
  Commutativité   : a + b = b + a | a · b = b · a
  Associativité   : (a+b)+c = a+(b+c) | (a·b)·c = a·(b·c)
  Distributivité  : a·(b+c) = a·b + a·c | a+(b·c) = (a+b)·(a+c)

Théorèmes de De Morgan

Théorèmes de De Morgan
a + b = a̅ · b̅   |   a · b = a̅ + b̅

→ Le complémentaire d'un OU est un ET de complémentaires, et inversement.

💡 Exemple de simplification

S = a · (b + c) + a · b

= a · b + a · c + a · b   (distributivité)

= a · b + a · c   (idempotence : b + b = b)

= a · (b + c)   (factorisation)

Formes canoniques

Forme canonique disjonctive (somme de produits)
S = Σ(indices des lignes où S=1) — on écrit le ET des entrées

💡 Forme canonique de la majorité

Lignes où S=1 : 3(a̅bc), 5(ab̅c), 6(abc̅), 7(abc)

S = a̅bc + ab̅c + abc̅ + abc

📕 4. Simplification par Karnaugh

La méthode de Karnaugh est une technique图形 pour simplifier les fonctions logiques. On regroupeles "1" adjacents.

Règles du tableau de Karnaugh
  • Former des groupes de 1, 2, 4, 8, 16… cases
  • Les groupes sont des puissances de 2
  • Les cases aux bords sont adjacentes (torique)
  • Les groupes doivent être les plus grands possibles
  • Une même case peut appartenir à plusieurs groupes

💡 Exemple : Table de Karnaugh à 3 variables

         ab
        00  01  11  10
    ┌────┬────┬────┬────┐
 c=0│ 0  │ 0  │ 1  │ 0  │
    ├────┼────┼────┼────┤
 c=1│ 0  │ 1  │ 1  │ 1  │
    └────┴────┴────┴────┘

  Groupe horizontal (c=1, b=1) : a = 1 → a·b
  Groupe vertical (b=1) : a·b + bc = b·(a+c)

⚠️ Méthode alternative : Quine-McCluskey

Pour les fonctions à 4+ variables ou avec des "don't care" (X), utiliser la méthode de Quine-McCluskey (algorithmique, programmable).

Les cases marquées X peuvent être considérées comme 0 ou 1 selon ce qui arrange !

📒 5. Circuits combinatoires usuels

Multiplexeur (MUX)

Multiplexeur 4→1
4 entrées de données (D0-D3), 2 entrées de sélection (S0-S1), 1 sortie Y
Y = D₀ si S=00, Y = D₁ si S=01, Y = D₂ si S=10, Y = D₃ si S=11

Démultiplexeur (DEMUX)

1 entrée → routing vers 1 parmi N sorties selon le sélecteur.

Codec encodeur / décodeur

🔄

Encodeur : N lignes d'entrée → log₂N lignes de sortie (binaire)

🔄

Décodeur : n lignes d'entrée → 2ⁿ lignes de sortie (un seul 1 actif)

➕

Additionneur : addition binaire (半加器, 全加器 avec retenue)

🔍

Comparateur : compare deux mots binaires (A > B, A = B, A < B)

💡 Additionneur complet (Full Adder)

Entrées : a, b, Rin (retenue entrante)

Sorties : S = a ⊕ b ⊕ Rin   |   Rout = (a·b) + (Rin·(a⊕b))

En cascade : addition de mots de n bits

⚠️ Circuits intégrés courants

• 74HC08 : 4 portes ET à 2 entrées

• 74HC32 : 4 portes OU à 2 entrées

• 74HC04 : 6 portes NON

• 74HC86 : 4 portes XOR

• 74HC151 : multiplexeur 8→1
