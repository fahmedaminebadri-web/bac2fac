📘 1. Introduction et définitions

Le GRAFCET (GRAphe Fonctionnel de Commande Étape Transition) est un outil de modélisation graphique des systèmes automatisés séquentiels. Il a été créé en France en 1977.

Un GRAFCET est composé d'étapes, de transitions et de liaisons orientées.

Éléments de base
  ■  = Étape (state)      → représente une situation du système
  ━━ = Liaison orientée   → relie les étapes
  ┼  = Transition         → permet le passage d'une étape à l'autre
  a  = Réceptivité        → condition logique de franchissement
📍

Étape initiale : double carré ■ — état du système au démarrage

✅

Étape active : étape en fonctionnement (marquage)

🔗

Transition : barre horizontale — séparateur entre étapes

➡️

Réceptivité : condition écrite près de la transition

💡 Exemple simple

□──▶┼──▶□──▶┼──▶□

a b

• Étape 1 active → si a=1 → on franchit la transition → étape 2 active

• Étape 2 active → si b=1 → on franchit → étape 3 active

📗 2. Règles d'évolution (règles du GRAFCET)

Le GRAFCET évolue selon 5 règles fondamentales :

1️⃣

Règle 1 : À un instant donné, une étape est soit active, soit inactive

2️⃣

Règle 2 : Les transitions sont franchies quand l'étape en amont est active ET la réceptivité est vraie

3️⃣

Règle 3 : Le franchissement d'une transition active l'étape en aval et désactive l'étape en amont

4️⃣

Règle 4 : Plusieurs transitions peuvent être franchies simultanément

5️⃣

Règle 5 : Si une étape doit être.active et désactivée en même temps, elle reste active

⚠️ Franchissement d'une transition

Une transition est validée si TOUTES les étapes précédentes sont actives.

Une transition est franchie si elle est validée ET sa réceptivité est vraie.

→ Le franchissement est instantané et indissociable (atomic)

📙 3. Actions associées aux étapes

Chaque étape peut être associée à une ou plusieurs actions (commandes de préactionneurs, affichages…).

Représentation
     Étape 3
  ┌──────────┐
  │   3      │  —  action 1
  │  ────────│  —  action 2
  │ KM1 = 1  │  —  ordre
  └──────────┘
🔒

Action continue : exécutée tant que l'étape est active (KM1=1)

⬆️

Action impulsionnelle : exécutée brièvement au moment du franchissement

⏱️

Temporisation : action retardée ou limitée dans le temps (ex: A /t₁/ 5s)

🔢

Compteur : action qui incrémente/décompte un compteur

Syntaxe des temporisations
A /t₁/ 5s : action A, temporisée de 5 secondes après activation de l'étape
A /t₁/ : action A maintenue pendant la durée de l'étape

📕 4. Réceptivités (conditions de transition)

Une réceptivité est une condition logique (VRAI ou FAUX) qui permet de franchir une transition.

Types de réceptivités
a          : front montant du signal a (0→1)
/a         : front descendant (1→0)
a.b        : ET logique (a ET b)
a+b        : OU logique (a OU b)
/a.b       : a̅ ET b

💡 Exemples de réceptivités

S1.S2     → bouton Start ET fin de course S2

/S1        → front descendant de S1

cy12       → bit de transition，来自 une étape 12

t > 10s    → temporisation écoulée

📒 5. Structures particulières

Séquence linéaire

Étapes exécutées les unes après les autres : □ → □ → □ → …

Saut d'étapes (Aiguillage OU)

Selon la condition, on saute une séquence :

Aiguillage OU
         ┌─── a ───▶[3]──▶┐
    [2]─┤                 ├─▶[5]
         └─── b ───▶[4]───┘

Séquences simultanées (Aiguillage ET)

Plusieurs séquences qui s'exécutent en parallèle :

Aiguillage ET (parallélisme)
              [3]
               │
         ┌─a─ ─ ─ ─ ┐
    [2]─┤         ├──[5]
         └─ ─ ─ ─ ─┘
               │
              [4]

Séquences exclusives (X OU)

Une seule des branches s'exécute (choix entre deux séquences).

📓 6. Grafcet hiérarchique

Pour les systèmes complexes, on utilise la hiérarchisation :

🏗️

GRAFCET de niveau 1 (système) : description fonctionnelle, indépendante de la technologie

⚡

GRAFCET de niveau 2 (PO) : description technologique, avec les actionneurs et capteurs réels

🔄

Macro-étapes : représentation condensée d'une séquence

🔗

Forçages : une étape peut forcer l'état d'un autre GRAFCET

💡 Exemple : Commande de moteur

□──▶┼──▶□(KM1=1)──▶┼(dcy)──▶□(KM2=1)──▶┼──▶□

• Étape 1 :初始 → attente de démarrage (dcy)

• Étape 2 : KM1 mis sous tension (rotation droite)

• Étape 3 : KM2 mis sous tension (rotation gauche)

• Étape 4 : arrêt

⚠️ Différence Grafcet / Ladder

• GRAFCET : modèle séquentiel — étapes, transitions, actions

• Ladder (API) : langage grafcet-like pour automates programmables

→ Le GRAFCET est d'abord un outil d'analyse et de conception
