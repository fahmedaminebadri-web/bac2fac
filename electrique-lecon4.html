📘 1. Définition et architecture

Un microcontrôleur est un circuit intégré qui contient sur une même puce : processeur (CPU), mémoire, et périphériques. C'est le "cerveau" des systèmes embarqués.

🧠

CPU : unité centrale — exécute les instructions

💾

Mémoire programme : stocke le code (Flash)

📦

Mémoire RAM : données temporaires

🔌

Périphériques : timers, ports, CAN, UART, I2C, SPI

💡 Exemples courants

• Arduino (ATmega328P) : 8 bits, 16 MHz, 32 KB Flash, 2 KB RAM, 14 E/S

• PIC16F877A : 8 bits, 20 MHz, 14 KB Flash, 368 B RAM, 33 E/S

• STM32 (ARM Cortex) : 32 bits, jusqu'à 180 MHz, 1 MB Flash, centaines d'E/S

📗 2. Organisation de la mémoire

Un microcontrôleur possède plusieurs espaces mémoire séparés :

Espaces mémoire
  Zone programme : adressée par le PC (compteur ordinal)
    ├── Flash / ROM : code permanent (programme)
    
  Zone données : adressée par le CPU
    ├── RAM : variables, pile (stack)
    ├── EEPROM : données persistantes
    └── Registres spéciaux (SFR) : configuration des périphériques

Registres

Les registres sont des cases mémoire spéciales qui pilotent le fonctionnement du microcontrôleur.

💡 Registres importants

• PORTx : données en entrée/sortie

• DDRx : direction (entrée ou sortie)

• TCNTx : compteur d'un timer

• OCRx : valeur de comparaison d'un timer

📙 3. Ports d'entrée/sortie (GPIO)

Les GPIO (General Purpose Input/Output) permettent de lire des capteurs ou commander des actionneurs.

Configuration d'un port
DDRx = 0bXXXX_XSET → bits à 1 = SORTIE
DDRx = 0bXXXX_XCLR → bits à 0 = ENTRÉE
⬆️

Entrée numérique : lit 0 (LOW) ou 1 (HIGH) — bouton, capteur tout-ou-rien

⬇️

Sortie numérique : met 0V (LOW) ou Vcc (HIGH) — LED, relais, moteur

📊

Entrée analogique (CAN) : convertit une tension en valeur numérique (0-1023 pour 10 bits)

🔧

PWM : modulation de largeur d'impulsion — contrôle de vitesse moteur, luminosité LED

💡 Exemple : Commander une LED

1. Configurer le bit en sortie : DDRB |= (1 << PB0)

2. Allumer la LED : PORTB |= (1 << PB0)

3. Éteindre la LED : PORTB &= ~(1 << PB0)

📕 4. Timers et compteurs

Un timer est un compteur matériel qui s'incrémente à chaque impulsion d'horloge. Il permet de :

⏱️

Générer des bascules temporelles précises

🔄

Créer des signaux PWM

📏

Mesurer des durées entre deux événements

📡

Compter des impulsions externes

Timer 8 bits (ex: ATMega328)
Fréquence d'overflow : f_overflow = f_cpu / (256 × Prescaler)
Résolution : 8 bits → modulo 256

💡 Exemple

Timer 8 bits, quartz 16 MHz, prescaler 1024 :

f_overflow = 16 000 000 / (256 × 1024) ≈ 61 Hz

→ Une interruption toutes les ~16 ms

📒 5. Interruptions et communication

Interruptions

Une interruption est un mécanisme qui interrompt le programme principal pour exécuter un code urgent, puis reprend le programme.

🔔

Timer overflow : débordement du timer

🔔

External interrupt : changement sur une broche

🔔

UART RX/TX : donnée reçue/envoyée sur port série

🔔

ADC complete : conversion analogique terminée

Communication série (UART)

UART — Paramètres
Débit (baud rate) : 9600, 115200 bauds…
Format : 8 bits de données, 1 bit de stop, sans parité (typique)

⚠️ Autres bus de communication

• I²C : 2 fils, communication avec capteurs, écrans (lent)

• SPI : 3-4 fils, très rapide (capteurs, mémoires SD)

• CAN : automobile, industriel, très robuste (bruit)

📓 6. Programmation (Arduino / PIC)

Arduino (langage C/C++ simplifié)

Structure d'un programme Arduino
void setup() {
  pinMode(13, OUTPUT);  // configurer broche 13 en sortie
}

void loop() {
  digitalWrite(13, HIGH); // allumer LED
  delay(1000);            // attendre 1 seconde
  digitalWrite(13, LOW);  // éteindre LED
  delay(1000);            // attendre 1 seconde
}

PIC (assembleur ou C)

💡 Exemple en C pour PIC

TRISB = 0x00;         // PORTB en sortie

PORTB = 0xFF;         // Toutes les sorties à 1

while(1) { ... }      // Boucle infinie

⚠️ Bonnes pratiques

• Toujours initialiser les ports dans setup()

• Utiliser les interruptions pour les événements temporels critiques

• Déclarer les variables en volatile si utilisées en interruption

• Attention à l'overflow des compteurs (remise à zéro)
